Phase 2 Summary (Day 13 → Day 15): Keyword-based schema selection ➝ semantic retrieval (RAG)
OVERVIEW - Replaced keyword-based schema filtering with semantic
retrieval (RAG) - System now retrieves relevant tables using embeddings
before generating SQL

Day 13 — Embeddings - Enable semantic understanding of schema
        * Converted schema docs → embeddings
        * Source: schema_docs.json
Day 14 — Vector DB - Make schema searchable via similarity
        * Stored embeddings in Chroma (vector_store.py)
        * Enabled similarity search
Day 15 — Retrieval Pipeline - Replace keyword filtering with semantic retrieva
        * Retrieve relevant schema using embeddings
        * Build context from retrieved docs
        * Inject into LLM prompt (replacing schema filtering)    

------------------------------------------------------------------------

embeddings.load_schema_docs()
Loads schema metadata from JSON file. LLM cannot understand raw DB → needs descriptions.

embeddings.format_doc(doc)
Converts JSON → readable text (JSON → text) | Why needed: Embeddings work on text, not structured JSON
Example output
        Table: users
        Description: Stores user account information
        Columns: user_id, signup_date

embeddings.generate_embeddings(docs):
Converts each formatted doc → embedding vector (text → embedding vectors) --> Enables semantic similarity search
Output
[(text, embedding_vector)]

vector_store.get_collection():
Creates/loads Chroma collectionm --> Logical container for embeddings

vector_store.store_embeddings(collection, embedded_docs):
Stores embeddings into Chroman --> Makes schema searchable later

vector_store.query_collection(collection, query_embedding):
Finds most similar documents --> Core retrieval step of RAG
Input
embedding of user question
Output
{
  "documents": [[doc1, doc2]]
}

retriever.retrieve_relevant_docs(question):
Embeds user question → Calls query_collection() → Returns top relevant schema docs --> Encapsulates retrieval logic (clean separation)

context_builder.build_context(docs):
Combines retrieved docs into prompt text --> LLM needs structured, readable context
Output
        Relevant Database Schema:
        Table: trades ...
        Table: users ...

generate_sql.generate_sql(user_query):
Calls retriever → Builds context → Sends to LLM → Returns SQL --> Final step: converts NL → SQL using retrieved schema
